# 告警不产生问题诊断

## 问题现象
后端的告警管理、告警查看没有新的告警产生，但VP端在运行且有JSON数据传过来。

## 问题分析

### 1. 已修复的问题 ✅
**function_id 命名不匹配**
- **问题**：Kafka Consumer 调用的 function_id 与插件注册的 ID 不一致
  - Consumer 使用：`fall_detection`, `fight_detection`, `armed_detection`
  - 插件注册：`fall`, `fight`, `armed`
- **修复**：已统一为插件注册的ID格式

### 2. 当前主要问题 ⚠️
**VP端二级检测器未启用**

从 Kafka 消息中看到的数据结构：
```json
{
  "primary_label": "person",
  "sub_targets": [],         // ❌ 空数组
  "secondary_labels": [],    // ❌ 空数组  
  "track_id": -1            // ❌ 没有跟踪
}
```

### 3. 告警插件的数据要求

| 插件 | 需要的数据 | 当前状态 |
|-----|-----------|---------|
| **fall_detection** | `sub_targets` 中的 keypoints（17个关键点） | ❌ sub_targets 为空 |
| **fight_detection** | `sub_targets` 中的 "fight" 标签 | ❌ sub_targets 为空 |
| **armed_detection** | `sub_targets` 中的 "armed" 标签 | ❌ sub_targets 为空 |
| **phone_detection** | `sub_targets` 中的 "phone" 标签 | ❌ sub_targets 为空 |
| **knife_detection** | `sub_targets` 中的 "knife" 标签 | ❌ sub_targets 为空 |
| **stick_detection** | `sub_targets` 中的 "stick" 标签 | ❌ sub_targets 为空 |

## 解决方案

### 方案1：启用 VP 二级检测器（推荐）✅

需要在 VP 的配置中启用 Layer 5-6 检测器：

```cpp
// VideoPipe/TensorRT-YOLO/examples/VideoPipe/src/load_balance_impl.cpp

// 1. 启用姿态检测器（用于摔倒检测）
auto pose_detector = std::make_shared<vp_trt_pose_detector_node>(...);

// 2. 启用行为检测器（用于打架、持械等）
auto behavior_detector = std::make_shared<vp_trt_classifier_node>(...);

// 3. 在管道中添加这些检测器
pipe->vp_nodes_RT = {
    primary_detector,    // Layer 4: YOLO检测
    pose_detector,       // Layer 5: 姿态估计
    behavior_detector,   // Layer 6: 行为分类
    kafka_broker
};
```

#### VP端需要启用的检测器：

1. **姿态估计器（Pose Estimator）**
   - 用于：摔倒检测
   - 输出：17个COCO关键点（肩膀、臀部、膝盖等）
   - 模型：YOLOv8-pose 或类似模型

2. **行为分类器（Behavior Classifier）**
   - 用于：打架、持械、玩手机等检测
   - 输出：行为标签（fight, armed, phone, knife, stick等）
   - 模型：自定义行为分类模型

3. **跟踪器（Tracker）**
   - 用于：track_id 生成（告警去重、轨迹分析）
   - 算法：DeepSORT、ByteTrack等

### 方案2：临时测试方案（开发调试用）⚠️

如果暂时无法启用二级检测器，可以修改插件逻辑为"检测到person就告警"：

```python
# 临时修改 fall_detection.py（仅用于测试）
def fall_detection_plugin(frame, demarcate=None, snapshot=None, threshold=None):
    camera_id = frame.get("camera_id")
    
    for target in frame.get("targets", []):
        label = target.get("primary_label")
        
        # ⚠️ 临时逻辑：只要检测到person就触发告警
        if label == "person":
            logger.warning(f"[AI] 测试模式：检测到person，触发摔倒告警")
            
            event_data = {
                "camera_id": camera_id,
                "frame_index": frame.get("frame_index"),
                "track_id": target.get("track_id", -1),
                "primary_label": label,
                "x": target.get("x"),
                "y": target.get("y"),
                "width": target.get("width"),
                "height": target.get("height"),
                "confidence": 0.85,
                "alarm_type": "摔倒检测（测试）",
                "snapshot": snapshot
            }
            
            asyncio.create_task(save_event("摔倒事件", event_data))
            break  # 只触发一次
```

**注意**：方案2会产生大量误报，仅用于验证告警流程是否正常，不应用于生产环境！

## 数据流程图

```
VP端 Kafka Producer
  ↓
  ├─ Layer 4: YOLO检测 → primary_label: "person"
  ├─ Layer 5: 姿态估计 → sub_targets: [{keypoints: [...]}]  ❌ 当前未启用
  ├─ Layer 6: 行为分类 → sub_targets: [{label: "fight"}]   ❌ 当前未启用
  └─ Tracker → track_id: 123                               ❌ 当前未启用
  ↓
Kafka Topic: vp_behavior
  ↓
后端 KafkaBehaviorConsumer
  ↓
AI Dispatcher
  ↓
插件（fall_detection等）
  ↓
检查 sub_targets ❌ 为空，不满足告警条件
  ↓
❌ 不产生告警
```
