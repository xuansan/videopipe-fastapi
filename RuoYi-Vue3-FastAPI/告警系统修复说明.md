# 🚨 告警系统修复说明

## 问题根源
VP通过Kafka发送检测数据到 `vp_behavior` topic，但后端缺少对应的消费者来处理这些消息并触发告警。

---

## ✅ 已修复内容

### 1. 创建行为告警消费者
**文件：** `module_kafka_bridge/kafka_behavior_consumer.py`

**功能：**
- 监听VP的 `vp_behavior` topic
- 自动识别检测到的person目标
- 触发行为检测插件（摔倒、打架、持械、吸烟、玩手机等）
- 插件判断是否满足告警条件
- 满足条件时自动保存到数据库

### 2. 集成到系统启动
**文件：** `server.py`

**修改：**
- 启动时自动启动behavior consumer
- 关闭时自动停止consumer

## 📊 工作原理

```
VP (VideoPipe)
  ├── 检测到person
  ├── Kafka发送到vp_behavior
  │      ↓
后端 BehaviorConsumer
  ├── 接收Kafka消息
  ├── 识别person目标
  ├── 触发行为检测插件
  │   ├── fall_detection
  │   ├── fight_detection
  │   ├── armed_detection
  │   ├── smoking_detection
  │   └── phone_detection
  │      ↓
AI Plugin
  ├── 判断告警条件
  │   ├── 检查区域（demarcate）
  │   ├── 检查阈值（threshold）
  │   └── 检查持续时间
  │      ↓
  ├── ✅ 满足条件 → 保存数据库
  │      ├── sys_alarm (告警记录)
  │      └── sys_alarmview (告警视图)
  │      ↓
  └── 前端显示告警
```

## ⚙️ 配置说明

### 默认触发的行为检测
当检测到person目标时，系统会自动触发以下检测：
- `fall_detection` - 摔倒检测
- `fight_detection` - 打架检测
- `armed_detection` - 持械检测
- `smoking_detection` - 吸烟检测
- `phone_detection` - 玩手机检测

### 调整触发策略
修改 `kafka_behavior_consumer.py` 第94-100行：
```python
behavior_types = {
    'fall_detection',      # 可以注释掉不需要的
    'fight_detection',
    # 'armed_detection',   # 注释示例
    'smoking_detection',
    'phone_detection'
}
```

## 🐛 常见问题

### Q1: 告警页面还是没数据？
**检查清单：**
1. ✅ 后端日志有 `[BehaviorConsumer] 收到Kafka消息` 吗？
2. ✅ 后端日志有 `触发告警: fall_detection` 吗？
3. ✅ 插件是否正确加载？检查 `[PluginLoader]` 日志
4. ✅ 数据库中 `sys_alarm` 表有数据吗？

### Q2: Kafka消费者启动失败？
**可能原因：**
- Kafka服务未启动
- Kafka端口不是9092
- Topic不存在

**解决方案：**
```bash
# 检查Kafka
systemctl status kafka

# 创建topic（如果不存在）
kafka-topics.sh --create --topic vp_behavior \
    --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1
```

### Q3: 插件未找到？
检查插件是否正确加载：
```python

# 确认插件列表包含新增的5个插件
```

## 📈 性能优化

### 减少不必要的检测
如果摄像头很多，可以：
1. 只触发特定类型的检测
2. 增加检测间隔
3. 添加ROI区域过滤

### 示例：每5帧触发一次
```python
# 在kafka_behavior_consumer.py中添加
frame_index = frame_data.get('frame_index', 0)
if frame_index % 5 != 0:  # 每5帧触发一次
    continue
```
